# 贡献指南

感谢你对这个项目感兴趣！

## 🤝 如何贡献

### 报告问题

如果你发现了 bug 或有功能建议：

1. 检查 [Issues](https://github.com/Aster-Letter/Aster-Letter.github.io/issues) 是否已有类似问题
2. 如果没有，创建新的 Issue
3. 清楚地描述问题或建议
4. 如果是 bug，请提供复现步骤

### 提交更改

1. Fork 这个仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

## 📝 代码规范

### HTML
- 使用语义化标签
- 保持良好的缩进（2 或 4 个空格）
- 添加必要的注释

### CSS
- 使用有意义的类名
- 遵循 BEM 命名规范（可选）
- 保持样式的组织性

### JavaScript
- 使用现代 ES6+ 语法
- 添加注释说明复杂逻辑
- 保持代码简洁易读

## 🎯 贡献方向

### 欢迎的贡献类型

- 🐛 Bug 修复
- ✨ 新功能
- 📝 文档改进
- 🎨 样式优化
- ♿ 可访问性改进
- 🌐 多语言支持
- ⚡ 性能优化

### 特别欢迎

- 增加新的页面模板
- 改进响应式设计
- 添加动画效果
- 优化移动端体验
- 完善文档和教程

## 💡 建议

- 保持更改简洁明了
- 一次 PR 只解决一个问题
- 确保代码在不同浏览器中都能正常工作
- 测试你的更改

## 📧 联系

如有任何问题，欢迎：
- 创建 Issue
- 发送邮件到项目维护者

再次感谢你的贡献！🎉
